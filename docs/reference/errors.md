# 错误处理

当接口返回错误时，先看 HTTP 状态码，再看响应体里的错误信息。

## 常见状态码

| 状态码 | 含义 | 排查方向 |
| --- | --- | --- |
| `400` | 请求参数错误 | 检查 JSON 格式、模型名、messages 格式 |
| `401` | 鉴权失败 | 检查 API Key 是否正确、是否带了 Bearer |
| `403` | 无权限 | 检查账号权限、额度或模型访问权限 |
| `404` | 资源不存在 | 检查接口路径和模型 ID |
| `429` | 请求过于频繁 | 降低并发，增加重试退避 |
| `500` | 服务端错误 | 稍后重试，必要时联系管理员 |

## 常见问题

### 提示 Unauthorized

检查请求头是否正确：

```http
Authorization: Bearer your-api-key
```

注意 `Bearer` 后面有一个空格。

### 提示模型不存在

请回到控制台确认模型 ID。模型 ID 需要和控制台展示完全一致。

### 请求一直超时

可以从三方面排查：

- 当前模型是否响应较慢
- 请求内容是否过长
- 你的服务端网络是否能访问 `api.mugua.link`

### 前端跨域报错

不要在浏览器里直接暴露 API Key。请改成通过你的后端服务转发请求。
