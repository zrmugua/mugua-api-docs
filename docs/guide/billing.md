# 额度与计费

额度、价格和可用模型以控制台展示为准。调用前建议先确认账号余额、API Key 状态和目标模型是否可用。

## 查看余额

打开控制台：

```text
https://api.mugua.link
```

在控制台中查看账号余额、额度记录和 API Key 状态。

## 调用消耗

一次请求通常会根据输入和输出 token 计费。具体消耗取决于：

- 使用的模型
- 输入内容长度
- 输出内容长度
- 是否开启流式输出
- 是否使用图片、工具调用等扩展能力

## 上线建议

正式上线前建议完成这些检查：

- 为生产环境单独创建 API Key
- 在服务端保存 API Key
- 给用户请求增加限流
- 记录请求失败日志
- 对 `429`、`500` 等错误做重试或降级处理

## 额度不足

如果接口返回额度不足、余额不足或 `429` 相关错误，请先检查：

- 账号余额是否充足
- API Key 是否仍然有效
- 当前模型是否有可用额度
- 是否短时间内请求过多
